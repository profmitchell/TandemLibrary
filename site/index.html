About Tandem

Tandem is a collaborative MIDI canvas for building custom control surfaces, sharing presets, and syncing multiple performers.

Focus Mode

Precision control for a single component. Ideal for automation, MIDI learn, and quick tweaks while recording.

Canvas Mode

Drag, drop, and build full control surfaces with knobs, sliders, pads, and XY controls. Save layouts as presets for reuse.

Real-Time Collaboration

Sync multiple devices for shared control sessions. Great for live sets, band rehearsals, and studio workflows.

Theme Studio

Swap visuals instantly, import community themes, and export your own palettes for collaborators or clients.

Getting started

Import-ready files with a clean handoff from the Tandem app to the library and back.

1. Download a file

Grab a theme (.tandemt) or config (.tandemc) and keep the preview image for reference.

2. Open in Tandem

Use Files, AirDrop, or share sheet. Tandem can auto-import or ask for approval depending on your settings.

3. Find it fast

Imported themes surface in Theme Manager, and configs appear in the preset manager and settings.

Presets & Imports

Everything in the library is designed to import cleanly into the Tandem app with validation and clear metadata.

.tandemc configs

Complete workspace layouts, MIDI mappings, and defaults packaged as human-readable YAML/JSON. Great for sharing full rigs.

.tandemt themes

Color palettes and UI look/feel packages. Import a single theme or a full pack to match a show or studio vibe.

Import flow

Open a file from Files, AirDrop, or a share sheet and Tandem will import it in the background. You can disable auto-import in Settings for manual approvals.

Compatibility checks

Imports are validated on-device with warnings surfaced when data needs attention, keeping your library safe to use.

How items are discovered

The build script scans /themes and /configs, pairs each file with a preview image (matching slug), and reads optional metadata.

Metadata support

Themes can include metadata.title, metadata.description, and metadata.tags. Configs can include metadata.title and metadata.description.

Download + source

Download buttons use GitHub raw file URLs, and View Source links point to the file in the repo.
